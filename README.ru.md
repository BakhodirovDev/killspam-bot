## Что это такое

Бот для модерации Telegram-групп в сфере IT и обучения. Любой желающий может добавить его в свою группу, выполнить команду `/enable` и мгновенно получить защиту. Бот является **мультиарендным (multi-tenant)** и работает по принципу **«свой ключ» (BYOK)**: каждая группа подключает собственный ключ Google Gemini для ИИ-модерации, что исключает общие расходы и риск утечки центрального ключа. Даже без ключа бесплатный уровень на основе правил продолжает защищать группу.

## ✨ Возможности

- **Мультиарендность и самообслуживание** — добавьте бота, введите `/enable` в группе, и готово. Админы каждой группы получают свои уведомления.
- **ИИ-модерация через BYOK** — команда `/setkey` открывает одноразовую HTTPS-форму для сохранения ключа Gemini группы (шифруется при хранении). Если ключа нет — работают правила по ключевым словам.
- **Обнаружение контента 18+** — ИИ-слой помечает сексуализированный контент и явные флирт-ловушки от спам-ботов, при этом настроен так, чтобы не трогать обычное общение, рекламу и флуд.
- **Сканирование фото профиля на наготу** — фото профиля новых участников проверяются локально с помощью NudeNet (без облачных запросов). Если фото явно непристойное — аккаунт банится, а его сообщение удаляется.
- **Защита от вредоносного ПО и адалт-ссылок** — ссылки на исполняемые файлы (`.apk`, `.exe`, …) или ссылки на каналы 18+ в описании профиля ведут к немедленному бану (проверяется до ключевых слов).
- **Проверка при вступлении и по первому сообщению** — профили проверяются как при вступлении участника, так и при его первом сообщении, что защищает и публичные группы с доступом по ссылкам.
- **Обнаружение скоординированных атак** — одинаковые сообщения от нескольких аккаунтов за короткий промежуток времени удаляются, а аккаунты банятся одновременно.
- **Дифференцированные меры** — за серьезные нарушения следует бан + удаление; за менее очевидные — временный мут на 24 часа с возможностью восстановления и проверкой админом. Админы могут забанить или размутить пользователя прямо из уведомления.
- **Инструменты для администраторов** — команды `/ban`, `/mute`, обучение ключевым словам, пометка аккаунтов и обратная связь по ложным срабатываниям — всё прямо в Telegram.
- **Отчеты об использовании и активности** — `/tokens` показывает использование токенов Gemini и стоимость; `/stats` показывает список групп и пойманный спам; оператор получает утренний отчет.
- **Чистота по умолчанию** — собственные уведомления бота и сообщения с командами админов автоматически удаляются из группы через несколько секунд.

## 🧠 Как это работает

Каждое сообщение в защищенной группе проходит через конвейер проверок от простых к сложным; первое срабатывание определяет результат:

1. **Проверка профиля (единожды для каждого участника)** — при вступлении и при первом сообщении бот проверяет описание профиля (bio) и фотографии.
   - Ссылка на исполняемый файл, ссылка на канал 18+ или явно непристойное фото (NudeNet) → **бан + удаление** (критично, приоритет над всем остальным).
   - Сомнительные фото **не вызывают действий** — бот реагирует только на однозначный вердикт, чтобы избежать мута реальных пользователей.
2. **Уровень ключевых слов (бесплатно)** — небольшой список регулярных выражений с высокой точностью (`patterns/seed_patterns.txt`) плюс ключевые слова, добавленные админами групп.
3. **Уровень ИИ (BYOK, опционально)** — если в группе установлен ключ Gemini, «чистое» сообщение классифицируется моделью Gemini 2.5 Flash на предмет спама с подтекстом 18+ или флирт-ловушек. Действует ограничение частоты запросов для контроля расходов.
4. **Детектор массовых рассылок** — идентичные сообщения от 3 и более аккаунтов в течение 2 минут расцениваются как скоординированная атака.

Обнаруженный спам удаляется; отправитель мьютится (на 24 часа, с возможностью восстановления) или банится (за серьезные нарушения / бот-сети), а админы группы получают уведомление с кнопками «Бан» / «Размутить». Все данные изолированы в рамках группы, и бот полностью молчит в группах, где не была выполнена команда `/enable`.

**Конфиденциальность:** ключ Gemini каждой группы шифруется при хранении с помощью Fernet; общего ключа не существует. Тексты отчетов о спаме и данные об использовании удаляются через 90 дней.

## 🚀 Установка и деплой

Бот работает как одиночный воркер на long-polling плюс небольшой HTTPS-эндпоинт для формы ввода ключа. Он спроектирован для [Railway](https://railway.app/), но подойдет любой хостинг, поддерживающий Python-воркер + PostgreSQL.

### 1. Создайте бота
В Telegram напишите [@BotFather](https://t.me/BotFather): `/newbot`, затем **отключите режим приватности** (`/setprivacy` → Disable), чтобы бот мог читать сообщения в группах.

### 2. Клонирование и установка
```bash
git clone <your-fork-url>
cd spam-bot
pip install -r requirements.txt
```

### 3. Настройка окружения
Скопируйте `.env.example` в `.env` и заполните его:

| Переменная | Обязательно | Описание |
|---|---|---|
| `BOT_TOKEN` | ✅ | Токен Telegram-бота от @BotFather |
| `DATABASE_URL` | ✅ | Строка подключения к PostgreSQL |
| `ADMIN_TELEGRAM_IDS` | ✅ | ID операторов через запятую (системные алерты + `/stats`) |
| `KEY_ENCRYPTION_SECRET` | ✅ | Ключ Fernet для шифрования ключей Gemini. Создать: `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"` |
| `BASE_URL` | ✅ | Публичный HTTPS URL бота (для создания ссылки `/setkey`) |
| `GEMINI_PRICE_IN` / `GEMINI_PRICE_OUT` | ➖ | $/1M токенов для оценки стоимости в `/tokens` (по умолчанию `0.30` / `2.50`) |
| `REPORT_HOUR` | ➖ | Час (Asia/Tashkent) для утреннего отчета (по умолчанию `9`) |
| `MAX_GROUPS_PER_OWNER` | ➖ | Защита от злоупотреблений: макс. групп на одного не-оператора (по умолчанию `20`) |

> ⚠️ Никогда не фиксируйте реальный `.env` в репозитории. Он добавлен в gitignore — пусть так и остается.

### 4. Деплой
На Railway: создайте проект, добавьте плагин PostgreSQL, установите переменные выше и запустите деплой. Команда запуска (см. `Procfile`):
```bash
python init_db.py && python -m spam_bot.main
```
`init_db.py` создает или мигрирует таблицы при каждом запуске. Локально можно использовать эти же две команды.

### 5. Защита группы
1. Добавьте бота в свою группу и сделайте его **админом** с правами **Удаление сообщений** + **Блокировка пользователей**.
2. Выполните `/enable` внутри группы.
3. (Опционально, для ИИ) Выполните `/setkey` и перейдите по одноразовой ссылке, чтобы сохранить ключ Gemini для группы.

### Откат (Rollback)
Каждый релиз помечается git-тегом (`vX.Y.Z`). Чтобы откатиться, передеплойте предыдущий тег; схема БД аддитивна, поэтому старый код будет работать с новой БД.

## 💬 Команды

| Команда | Кто | Что делает |
|---|---|---|
| `/enable` · `/disable` | Админы группы | Включение/выключение защиты в группе |
| `/setkey` | Админы группы | Сохранение ключа Gemini (приватная одноразовая ссылка) |
| `/ban` · `/mute` | Админы группы | Модерация пользователя, на чье сообщение дан ответ |
| `/tokens` | Админы / оператор | Использование токенов Gemini + стоимость (вчера / 7д / 30д) |
| `/stats` | Оператор (в ЛС) | Список групп + активность по спаму/банам |
| `/help` · `/privacy` | Все | Руководство пользователя / политика данных |
